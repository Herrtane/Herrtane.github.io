## Linux Device

리눅스 장치는 일반적으로 물리적 하드웨어에 대한 인터페이스를 제공하는 "/dev"에 저장된 특수 파일이다. 주요 장치 유형은 크게 3가지로 나뉜다.

- 문자 장치: 예를 들어 시스템 RAM, 그래픽 카드 버퍼, 터미널, 입력 주변 장치 등
- 블록 장치: 예를 들어 하드 디스크, 시스템 및 데이터 파티션 등
- 의사 장치: 물리적 장치에 해당하지 않는 장치, 예를 들어 /dev/random, /dev/null 등

## sysfs

sysfs 파일 시스템은 리눅스 커널에서 시스템 및 장치의 정보를 사용자 공간에 노출하기 위해 사용되는 가상 파일 시스템이다. OS 시간에 배웠듯이 Linux는 파일 단위로 장치를 관리하는 특이한 방식을 사용한다. 

<br/>

sysfs는 커널 객체의 속성을 파일 형태로 표현하여 사용자나 애플리케이션이 쉽게 접근하고 조작할 수 있도록 한다. 이를 통해 시스템 관리자나 개발자가 시스템의 상태를 모니터링하고 조정하는 데 유용한 도구이며, 커널과 사용자 공간 간의 상호작용이 보다 용이해진다. sysfs의 주요 특징과 구조는 다음과 같다.

- **구조**: sysfs는 `/sys` 디렉토리 아래에 계층 구조로 구성된다. 이 계층 구조는 커널 객체 모델을 반영하며, 장치, 버스, 클래스, 드라이버 등의 정보를 포함한다.
  - `/sys/class/`: 장치 클래스별로 정리된 디렉토리. 예를 들어, 네트워크 인터페이스는 `/sys/class/net/` 아래에 위치.
  - `/sys/devices/`: 실제 장치 트리 구조에 따라 정리된 디렉토리. 시스템의 물리적 장치들이 이 경로 아래에 나열된다.
  - `/sys/bus/`: 시스템 버스별로 정리된 디렉토리. 예를 들어, PCI 버스에 연결된 장치들은 `/sys/bus/pci/` 아래에 위치한다.
  - `/sys/block/`: 블록 장치들이 정리된 디렉토리. 예를 들어, 하드 드라이브와 같은 장치들이 이 경로 아래에 위치.
- **파일**: sysfs의 각 디렉토리에는 다양한 속성 파일이 존재하며, 이 파일들은 장치나 커널 객체의 속성을 나타낸다. 예를 들어, 네트워크 인터페이스의 속성 파일은 `/sys/class/net/eth0/` 아래에 위치하며, `speed`, `address`, `operstate` 등의 파일을 통해 해당 인터페이스의 속성을 확인할 수 있다.
- **읽기/쓰기**: 대부분의 sysfs 파일은 읽기 전용이지만, 일부 파일은 쓰기 가능하여 시스템 설정을 변경할 수 있다. 예를 들어, CPU의 주파수를 조절하는 파일은 쓰기 가능할 수 있다.
